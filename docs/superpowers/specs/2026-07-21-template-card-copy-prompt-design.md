# 模板卡片快速复制提示词设计

## 1. 目标

首页和模板库卡片底部当前展示收藏数量与创建日期。创建日期对视觉模板浏览和使用决策帮助有限，而且“最新”排序已经承担时间发现能力。

本次优化从卡片中隐藏日期标签，并将该位置替换为“复制”按钮，让用户无需进入详情页即可获得一条可直接使用的完整提示词。

## 2. 范围

- 只移除 `TemplateCard` 中的日期展示和日期格式化函数。
- 保留 `createdAt` 字段、API 返回、数据库数据、后台展示和排序能力。
- 首页与模板库复用同一个 `TemplateCard`，因此同步生效。
- 不改变卡片图片、标题、绿色标签、收藏按钮或详情页入口。
- 不改变详情页已有的变量编辑和复制能力。

## 3. 采用方案

卡片右下角使用带复制图标的“复制”按钮替换日期胶囊。按钮复制按模板变量默认信息渲染出的完整 Prompt，而不是带 `{{variable}}` 占位符的骨架。

该方案比复制原始骨架更直接，也比点击后进入详情页更符合“复制”按钮的明确承诺。

## 4. 提示词生成规则

为每个模板变量按以下优先级选择值：

1. 非空 `defaultValue`；
2. 第一个非空 `suggestions` 值；
3. 第一个非空 `options` 值；
4. 没有可用值时留空。

使用共享包现有的 `renderPromptTemplate()` 渲染最终 Prompt，确保占位符清理和标点规范与详情页一致。

复制前检查所有必填变量。如果任一必填变量经过上述回退后仍为空，则不写入剪贴板，并提示用户进入详情页补充变量。可选变量允许为空，由共享渲染器安全移除对应占位符。

将值推导与 Prompt 渲染封装为独立纯函数，使卡片组件只负责点击交互和反馈，便于单元测试与未来复用。

## 5. 交互行为

- 复制按钮是卡片链接内部的独立操作，点击时必须执行 `preventDefault()` 与 `stopPropagation()`，不得进入详情页。
- 成功复制后：
  - Toast 显示“提示词已复制”；
  - 当前卡片按钮短暂显示“已复制”；
  - 约 1.5 秒后恢复“复制”。
- 复制失败后：
  - Toast 显示“复制失败，请进入详情页手动复制”；
  - 按钮保持“复制”，不得显示虚假成功状态。
- 必填变量缺失时：
  - Toast 显示“该模板需要补充变量，请进入详情页”；
  - 不调用剪贴板 API。
- 连续点击时，每次复制结果一致；组件卸载时清理按钮恢复定时器。

## 6. 视觉与可访问性

- 按钮沿用收藏按钮的圆角胶囊、浅灰背景和边框风格，保持卡片底部视觉平衡。
- 使用项目现有 `IconCopy` 图标，文案为“复制”；成功时文案为“已复制”。
- 按钮设置动态 `aria-label`：默认“复制「模板名称」的提示词”，成功后“已复制「模板名称」的提示词”。
- 保留清晰的 hover 与键盘焦点状态。
- Toast 容器已有 `aria-live="polite"`，继续使用现有 `useToast()`，不增加另一套反馈机制。

## 7. 错误与兼容性

- 使用 `navigator.clipboard.writeText()`；异常统一进入失败反馈。
- 不因为剪贴板失败而跳转详情页，避免意外导航。
- 不记录、修改或回写模板 Prompt 数据。
- 日期数据仍可继续用于“最新”排序和未来的详情页、后台或审计功能。

## 8. 测试设计

新增测试覆盖：

1. 卡片源码不再渲染 `createdAt`，但 `PromptTemplate` 类型仍保留该字段。
2. 默认值优先于推荐值和严格选项。
3. 缺少默认值时依次回退到推荐值、严格选项。
4. 可选空变量可以安全渲染。
5. 必填变量没有可用值时返回不可复制结果。
6. 点击复制按钮阻止卡片导航。
7. 成功与失败使用不同 Toast 文案，只有成功才显示“已复制”。
8. 卡片继续保留收藏按钮、标题和标签。

验证包括纯函数测试、Web 回归测试、Lint、生产构建和真实页面检查。

## 9. 完成条件

- 首页和模板库卡片不再显示日期。
- 日期数据和排序功能没有删除或改变。
- 卡片可以复制一条没有未解析占位符、可直接使用的完整 Prompt。
- 必填变量缺失、剪贴板失败时不会产生错误复制或错误成功提示。
- 复制操作不会打开模板详情页，也不影响收藏操作。
