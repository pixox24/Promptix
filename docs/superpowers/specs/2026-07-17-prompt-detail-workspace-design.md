# Prompt 详情页首屏工作台设计策划

## 目标

把详情页首屏重组为约 16:9 的横向工作台，让用户在 1440×900 的常见桌面视口中一次看全封面图、元信息、变量填写和主要操作，不需要先滚动页面才能完成核心任务。

## 设计判断

当前页面的核心问题不是内容过多，而是三块内容彼此纵向叠加：封面使用 3:4 竖向比例，元信息独立落在封面下方，变量和操作又在右栏继续向下生长。结果是视觉焦点松散、首屏利用率低，且“看模板”和“使用模板”像两个断开的区域。

采用“沉浸式画布 + 工具面板”的工作台结构：左侧约 58% 承载横向视觉画布，模板信息与元数据作为画布上的分层内容；右侧约 42% 承载变量、Prompt 预览和操作。两栏共用统一高度与圆角外框，形成一个完整产品表面。

## 信息架构

- 左侧视觉画布：横向裁切图片、分类状态、收藏入口、标题、描述、标签、适用场景、变量/使用/收藏统计。
- 右侧变量面板：步骤提示、紧凑变量表单、可折叠 Prompt 预览、生成与辅助操作。
- 相似模板继续保留在工作台之后，作为次级浏览内容，不参与首屏核心任务。

## 视觉语言

- 基底使用暖灰白与深墨黑，品牌酸橙只用于状态、选择和主操作，避免整页平均用力。
- 工作台使用 20–24px 大圆角、细描边、分层阴影和轻微内高光，建立精致的实体感。
- 图片区使用真实图片与深色遮罩保证信息可读；不新增装饰性假素材。
- 控件改为 10–12px 圆角、44px 左右高度，选中态用酸橙浅底与深色文本，减少廉价的纯色块感。

## 响应式与可访问性

- `xl` 及以上启用统一高度横向工作台；1440×900 下首屏完整显示核心操作。
- 中小屏回落为单列自然流，不强制固定高度。
- 变量数量超过面板容量时，仅变量区内部滚动；底部主操作始终可见。
- 保留语义标题、标签、键盘焦点与错误提示；文本和控件保持足够对比度。

## 验收标准

1. 详情页存在单一 `prompt-detail-workspace`，桌面端使用约 `minmax(0, 1.35fr) / minmax(420px, .95fr)` 的横向布局。
2. 工作台桌面高度受视口约束，图片、元信息、变量和操作均在其中。
3. 左侧媒体区使用横向/全幅 `object-cover`，元信息以底部覆盖层呈现。
4. 右侧为纵向 flex 面板，变量区域可内部滚动，操作区固定在底部。
5. 1440×900 浏览器截图无水平滚动，首屏可见主生成按钮与完整左侧元信息。
