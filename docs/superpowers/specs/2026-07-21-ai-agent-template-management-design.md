# Promptix AI Agent 友好型模板管理设计

日期：2026-07-21
状态：设计已确认，待书面复核与实施计划
选定方向：智能分拣台（三栏治理工作区）

## 1. 背景与问题

当前模板管理页将搜索、状态筛选、精选切换、编辑、发布、下架和删除集中在同一张平铺表格中。随着模板数量、智能入库来源和语义分类维度增加，这种结构会产生以下问题：

- 模板没有按“需要完成的工作”组织，管理员只能在全量列表中寻找目标。
- 分类校对、精选运营、生命周期管理和内容编辑混在同一层级。
- 每行直接排列多个操作，信息密度高但任务优先级不清晰。
- 缺少批量选择、跨页处理、服务端分页和稳定排序。
- Agent 无法通过稳定的业务对象、风险规则和操作契约理解并管理模板，只能模拟人工点击或调用宽泛更新接口。
- 变更缺少批次、版本、解释、审批和回滚语义，无法支撑可控自动化。

现有系统已经具备可复用基础：模板生命周期、来源、封面、精选顺序、语义分类、分类置信度、未映射术语和分类审核状态。本设计在这些能力上增量建设，不推倒现有模型。

## 2. 目标与非目标

### 2.1 目标

1. 将模板管理定位为“批量治理中心”，优先支持批量分类、精选运营和生命周期管理。
2. 使用工作队列代替单一全量列表，让管理员直接处理待确认分类、重复模板、质量异常、精选候选和待审批事项。
3. 采用“定时巡检 + 临时自然语言指令”的 Agent 运行方式。
4. 允许 Agent 自动优化标题、摘要、公开分类和标签，同时保留完整版本与回滚能力。
5. 允许 Agent 在明确运营规则内自动调整精选；异常调整必须审批。
6. 发布、下架、永久删除以及其他高风险操作必须人工批准。
7. 为 Agent 提供结构化数据、受约束工具、显式风险等级和可验证执行流程。

### 2.2 非目标

- 不将模板管理页改造成纯聊天界面。
- 不以看板作为默认管理视图。
- 不让 Agent 直接写数据库或调用无约束的通用更新接口。
- 不在首期引入任意层级文件夹或复杂项目空间。
- 不改变 Promptix 现有后台品牌、主导航和基础视觉语言。

## 3. 核心设计原则

### 3.1 以任务组织，而不是以数据堆叠

默认入口展示 Agent 生成的工作队列。生命周期、来源、产物类型、场景和风格作为筛选维度，不再承担一级任务导航。

### 3.2 人与 Agent 共享同一事实模型

界面状态、API 状态和 Agent 工具返回值必须来自相同的稳定枚举与原因代码。中文文案是展示层映射，不能成为 Agent 的业务判断依据。

### 3.3 建议、审批、执行分离

Agent 先形成可验证的变更计划，再根据规则自动执行或提交审批。任何公开影响和破坏性操作都必须能解释、审计和回滚。

### 3.4 高密度但不拥挤

主列表承载批量操作所需字段；详细解释、预览和历史放入右侧检查器。低频与危险操作进入“更多”菜单，避免每行堆满按钮。

## 4. 信息架构

模板管理采用稳定三栏布局，并保留现有顶部后台导航。

### 4.1 左栏：工作队列

一级队列：

- `taxonomy_confirmation`：待确认分类
- `duplicate_candidates`：重复模板
- `quality_issues`：质量异常
- `featured_candidates`：精选候选
- `pending_approval`：待审批
- `failed_items`：失败与待重试

辅助视图：

- 来源：人工、文本优化、图片反推
- 生命周期：草稿、已发布、已下架
- 快捷筛选：高置信度优先、仅看未处理、包含质量问题
- 巡检状态：上次运行、下次运行、当前规则集

队列由服务端定义稳定 ID、计数逻辑和默认排序。管理员看到的名称可以调整，但 Agent 始终使用稳定 ID。

### 4.2 中栏：批量作业区

主列表字段：

- 选择框与封面缩略图
- 模板名称和来源
- 当前分类与建议分类
- 质量等级与 Agent 状态
- 生命周期与公开范围
- 更新时间

列表支持：

- 服务端分页、排序和组合筛选
- 当前页选择与“选择全部匹配结果”两种明确语义
- 跨页批量计划，不在前端保存大量模板 ID
- 上下文批量工具栏，仅显示当前范围可执行的操作
- 通过 URL 恢复队列、筛选、排序、页码和选中批次

模板行本身只保留主入口。编辑、查看历史、排除建议等低频操作收进右侧检查器或“更多”菜单。

### 4.3 右栏：预览与决策检查器

选中单个模板时显示：

- 封面与核心内容摘要
- 当前分类和 Agent 建议
- 修改前后差异
- 判断依据、置信度、风险等级
- 来源与历史版本
- 接受建议、跳过、加入批量

选中多个模板时显示：

- 选择范围与匹配条件
- 计划内自动项、审批项和冲突项数量
- 代表性差异与异常样本
- 创建批次、提交审批或调整范围

## 5. Agent 入口与运行方式

### 5.1 两类入口

1. 定时巡检：按当前规则集周期性检查新增与存量模板，生成 `AgentRun` 和工作队列。
2. 临时指令：管理员输入自然语言目标，例如“整理最近入库但未分类的海报模板”。

临时指令不能立即产生写操作。Agent 必须先返回：

- 对目标的结构化理解
- 查询范围与排除条件
- 使用的规则集
- 预计影响数量
- 风险分布
- 下一步是自动执行还是提交审批

### 5.2 标准执行链路

`理解目标 → 查询模板 → 生成计划 → 规则校验 → 展示差异 → 自动执行/等待审批 → 结果报告 → 可选回滚`

每次运行生成唯一 `runId`，每个变更计划生成唯一 `changeSetId`。同一计划重复提交必须保持幂等。

## 6. 权限与风险模型

### 6.1 可自动执行

- 标题、摘要、公开分类和标签优化
- 内部质量评分、重复检测和异常标记
- 草稿与已发布模板的结构化元数据整理
- 生成工作队列和推荐范围

这些操作仍需满足 Schema、版本和规则校验，并写入不可变版本记录。

### 6.2 规则内自动执行

Agent 可以自动调整精选状态与顺序，但必须同时满足：

- 固定精选名额或分类配额
- 最低质量与置信度门槛
- 单次最大变更比例
- 最小调整间隔
- 无异常流量或数据质量信号

突破任一约束时，整项或异常子集进入待审批队列。

### 6.3 必须审批

- 发布模板
- 下架模板
- 永久删除模板
- 修改 Prompt 骨架、变量定义或其他会改变模板生成行为的字段
- 突破精选规则的大规模调整
- 低置信度、规则冲突或异常影响的批量操作

永久删除采用二次确认，并要求输入明确删除原因；其回滚策略与普通内容版本回滚分开设计。

## 7. 领域模型

### 7.1 Template

保存稳定身份和当前指针：`id`、当前版本、生命周期、公开范围、精选状态、创建者和时间戳。

### 7.2 TemplateVersion

不可变内容快照，包含标题、摘要、描述、Prompt 骨架、负面提示词、变量、语义分类、标签、封面引用和来源信息。

### 7.3 AgentRun

表示一次定时巡检或临时任务，保存目标、结构化范围、规则集、状态、开始与完成时间、统计和错误摘要。

### 7.4 Proposal

表示单个模板的一组候选变化：

- `templateId`
- `baseVersion`
- `current`
- `proposed`
- `reasonCodes`
- `explanation`
- `confidence`
- `riskLevel`
- `requiresApproval`
- `validationResult`

### 7.5 ChangeSet

表示可执行批次，保存查询快照、显式排除项、提案集合、幂等键、预计影响、审批状态和执行结果。

### 7.6 RuleSet

保存自动化权限、精选配额、质量门槛、置信度阈值、批量上限和审批策略。规则集需要版本化，历史批次引用当时使用的具体版本。

### 7.7 Approval 与 AuditEvent

`Approval` 保存审批范围、审批人、决定和备注；`AuditEvent` 记录读取、建议、验证、执行、失败、重试和回滚事件。

## 8. Agent 工具契约

Agent 使用受约束工具，而不是通用 CRUD：

- `search_templates(filters, sort, cursor, fields)`
- `inspect_template(template_id, version?)`
- `validate_template(template_id, version?)`
- `plan_changes(goal, scope, ruleset_id, dry_run)`
- `preview_changes(change_set_id)`
- `execute_auto_changes(change_set_id, idempotency_key)`
- `submit_for_approval(change_set_id)`
- `get_change_set_status(change_set_id)`
- `rollback_change_set(change_set_id, expected_versions)`

契约要求：

- 输入输出使用 JSON Schema。
- 状态、分类和原因使用稳定枚举或 ID。
- 查询支持游标分页、字段选择和批量 ID。
- 所有写操作携带 `baseVersion` 或预期版本号。
- 计划接口默认 `dryRun`，执行接口只接受已验证的 `changeSetId`。
- 返回值明确区分成功、跳过、冲突、失败和需要审批。

## 9. 状态机与异常处理

### 9.1 批次状态

`analyzing → planned → auto_executing / awaiting_approval → partially_succeeded / succeeded / cancelled → rollback_available / rolled_back`

### 9.2 异常策略

- 版本冲突：停止覆盖，重新读取并生成新建议。
- 分类项不存在：保留原值并送入分类异常队列。
- Agent 输出不符合 Schema：服务端拒绝执行，记录原始响应和验证错误。
- 部分失败：保留独立成功项，失败项生成可重试子批次。
- 权限或规则变化：重新计算风险，旧审批不得继续使用。
- 回滚目标已被再次修改：显示冲突差异，禁止静默覆盖。
- 重复提交：通过幂等键返回原执行结果。

页面必须区分加载、无待处理项、筛选无结果、离线、权限不足和任务失败，不能统一显示空白状态。

## 10. 视觉与交互约束

- 继承现有 Promptix Admin 的深色顶部导航、浅色工作区和紫色强调色。
- 三栏在桌面端稳定存在；右栏允许收起，左栏允许窄化。
- 列表使用轻量分隔线和对齐建立层级，不将每行设计成独立卡片。
- 状态颜色必须同时配合文本或图标，不能仅靠颜色传达。
- 批量工具栏在选择后出现，并持续显示选择范围语义。
- 危险操作不得与常规编辑并列展示。
- 右栏差异优先显示有变化的字段，未变化字段默认折叠。
- Agent 解释保持简洁，并提供结构化原因代码供展开查看。

## 11. 验证策略

### 11.1 契约测试

- Agent 工具请求与响应 Schema
- 稳定枚举与原因代码
- 分页、排序和字段选择
- 无效或越权参数拒绝

### 11.2 规则与权限测试

- 自动执行与审批边界
- 精选配额、阈值、频率和最大变更比例
- 发布、下架、删除的强制审批
- 规则集版本变化后的审批失效

### 11.3 集成测试

- 幂等执行
- 乐观并发与版本冲突
- 部分失败和子批次重试
- 回滚成功与回滚冲突

### 11.4 端到端测试

- 定时巡检生成队列
- 临时指令生成计划
- 单条建议接受与跳过
- 跨页批量选择和排除
- 审批、执行、报告和回滚完整路径
- 空状态、错误状态与权限不足状态

## 12. 成功指标

- 每 100 条模板的平均治理耗时
- Agent 自动完成率
- 人工建议驳回率
- 执行异常率和重试成功率
- 回滚率及回滚原因分布
- 待处理队列平均滞留时间
- 精选调整后的使用与曝光效果
- Agent 解释被展开、调整或人工覆盖的比例

自动完成率不作为唯一目标。系统首先保证错误率可控、决策可解释、影响可预览、操作可回滚。

## 13. 验收标准

1. 管理员进入页面后可以直接看到按任务组织的队列，而不是无差别全量列表。
2. 管理员可以在列表中完成跨页批量选择，并明确知道作用范围。
3. 选择模板后，无需进入编辑页即可查看封面、Agent 建议、依据和差异。
4. 临时自然语言指令在任何写操作前都会生成结构化计划。
5. 标题、摘要、分类和标签可按规则自动修改，并产生版本与审计记录。
6. 精选只在规则范围内自动调整，异常调整自动进入审批。
7. 发布、下架和永久删除无法绕过人工审批。
8. 任何批次都能查看自动项、审批项、冲突项和失败项。
9. 幂等、版本冲突、部分失败和回滚行为均有自动化测试覆盖。
10. 人工界面、API 和 Agent 工具对状态与原因的表达一致。
