# 新模板分类确认默认勾选设计

## 目标

新模板进入草稿校对页面时，将“我已检查并确认产物类型、使用场景、风格和画面主体”复选框默认设为勾选，减少人工确认时的重复操作。

## 行为范围

- 仅修改新模板草稿校对组件的本地初始状态。
- 页面首次展示时复选框默认勾选。
- 管理员仍可手动取消勾选。
- 保存时继续按复选框的实时状态提交 `taxonomyConfirmed`。
- 不修改数据库默认值，不覆盖任何已保存模板的确认状态。
- 模板保存后若分类字段发生修改，继续沿用现有逻辑重新标记为待确认。

## 实现方案

将 `TemplateDraftReview` 中 `confirmed` 状态的初始值由 `false` 调整为 `true`。不修改 API 请求结构、后端路由或数据库结构。

## 测试与验收

- 新模板草稿校对页面首次渲染时，确认复选框处于勾选状态。
- 管理员可以取消勾选，取消后保存提交 `taxonomyConfirmed: false`。
- 保持现有分类完整性校验：默认勾选时，如果四类分类不完整或仍有待处理词，不能保存为已确认。
- 相关前端测试、类型检查和构建通过。
