# 模板变量推荐值设计与实施方案

> 状态：已实施并验证
>
> 日期：2026-07-18
>
> 范围：智能入库、模板数据契约、后台校对、图片详情页变量工作台

## 1. 目标

在不限制用户自由表达的前提下，为图片模板变量提供可直接点击填入的推荐值，降低首次使用成本，并让 AI 入库、后台校对、公开模板 API、草稿恢复和生成任务使用同一份变量定义。

## 2. 核心语义

变量中的 `options` 与 `suggestions` 必须严格区分：

- `options` 是封闭集合，仅供 `select`、`ratio` 使用。公开生成 API 必须拒绝集合外的值。
- `suggestions` 是快捷输入，仅供 `text`、`number` 使用。用户可以选择推荐值，也可以自由输入其他合法值。
- `image` 不允许配置普通推荐值，继续接受图片 URL。
- 为兼容历史模板，前端在 `text.suggestions` 缺失时可临时把旧 `text.options` 当作推荐值显示；新模板不得继续用这种写法。

## 3. 生成数量

AI 入库遵循以下规则：

| 类型 | 字段 | 数量 | 约束 |
|---|---|---:|---|
| `text` | `suggestions` | 4-6 | 简短、具体、可直接进入 Prompt、彼此显著不同 |
| `number` | `suggestions` | 3-5（仅有帮助时） | 使用字符串表示，与字段单位一致 |
| `select` | `options` | 4-8 | 封闭选项，必须覆盖常见且有差异的选择 |
| `ratio` | `options` | 3-5 | 从产品支持的标准比例中选择 |
| `image` | 无 | 0 | 不生成 `options` 或 `suggestions` |

数量是生成质量目标，不作为所有人工模板的发布硬门槛。后台允许管理员删改，避免为了满足数量而保留低质量内容。

## 4. 内容质量规则

- 单个值去除首尾空白后长度为 1-60 个字符。
- 同一数组内不得有完全重复的值。
- 推荐值不得包含“请输入”“可以尝试”等操作说明。
- 推荐值不得是完整 Prompt 或解释段落。
- `defaultValue` 应属于严格 `options`；自由输入变量的默认值可以不属于 `suggestions`。
- 推荐内容必须与变量 `label`、模板用途和 Prompt 占位符语义一致。
- 比例由系统支持集合控制，模型不得创造如“超宽电影比例”等不可解析值。

## 5. 数据流

```text
智能入库系统提示词
  -> 模型输出 TemplateDraft
  -> shared schema 清理与校验
  -> 后台校对 options/suggestions
  -> JSONB variables 保存
  -> 公开模板 API 原样返回
  -> VariableWorkbench 渲染
  -> 用户点击或自由输入
  -> 同一 values 状态
  -> Prompt、草稿、公开生成 API
  -> 后端再次执行严格选项校验
```

## 6. 后台交互

每个变量显示基本属性、严格选项和推荐值两个独立编辑区：

- 严格选项：用于 `select` / `ratio`，逗号分隔。
- 推荐值：用于 `text` / `number`，逗号分隔。
- 编辑时实时去除首尾空白和空项。
- 保存时由共享 schema 统一检查类型、重复项、长度和最大数量。
- 校验失败时阻止保存，避免非法变量进入公开模板。

## 7. 前端交互

- 推荐值显示在输入框下方，单行横向滚动，不改变页面主体布局。
- 点击推荐值与手工输入调用同一个 `onChange(key, value)`。
- 当前输入与推荐值完全相等时高亮对应标签。
- 再次点击当前值不需要特殊清空行为，避免误操作。
- 推荐值不改变必填校验、Prompt 行内编辑、草稿、复制或生成链路。

## 8. 数据库与兼容策略

`variables` 使用 JSONB，不增加数据库字段。新增迁移仅更新智能入库的系统默认提示词：

- 仅当数据库内容仍等于旧版内置默认值时升级。
- `updated_by` 非空或内容已被管理员修改时不覆盖。
- 已发布模板无需迁移，旧 `text.options` 由前端兼容读取。

## 9. 测试与验收

- shared：接受合法推荐值；清理空白；拒绝重复、超长、超量和错误变量类型。
- shared：自由文本不受推荐值限制；严格选项仍拒绝集合外值。
- 系统提示词：两条流程都明确包含 `suggestions`、数量及类型规则。
- 迁移：只升级旧默认提示词，不能无条件覆盖管理员内容。
- Web：推荐值来源优先级、选中态和点击填充逻辑具备契约测试。
- 全仓执行 `npm test`、`npm run build`、`npm run lint` 和 `git diff --check`。
